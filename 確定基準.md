# 確定基準

最終更新: 2026-06-23  
位置づけ: **確定済みルール・数値基準・合意済み設計方針**を1か所に集約する索引文書。  
詳細本文（現行の詳細記述）は `legacy/` を参照すること。PL 向け正本は `docs/` に整備する。本書は重複を避けた要約であり、**内容が食い違う場合は `docs/` を優先**する。

## 2. ゲームの骨格（確定）

- **作品名**: 想成のスコーリア（*Eidopoiesis*）
- **ジャンル**: 錬金工房と町が日々成長する継続型キャンペーンTRPG
- **運用**: 紙運用（トランプ山札・手書きシート）。4人パーティを基本難易度の前提とする
- **1日の流れ**: 朝 → 昼 → 夕 → 夜（`legacy/day_cycle.md`）
- **基本循環**: 町/工房の発展 → 新地景開放 → 探索 → 素材獲得 → 分解/作成/納品 → 発展
- **正本**: PL 向けルール `docs/`、開発参照 `legacy/`、ゲームデータ `catalog/`（SQLite）、公開 GitHub Pages

## 3. 判定システム（確定）

詳細: `legacy/dice_and_checks.md`

### 3-1. 基本構造

1. ダイスプール構築（原則 **d10 を1個**から開始）
2. 全ダイスを振る
3. 反応ループ（反応可能なダイスを任意順で処理）
4. 澱み判定
5. 達成値合計 → 目標値以上で成功

### 3-2. 属性ダイス

| 属性 | ダイス | 反応条件 | 反応効果 |
| --- | --- | --- | --- |
| 火 | d4 | 3以上 | 延焼: 自身→4、d4を1個追加 |
| 地 | d6 | 6 | 変動: 他ダイスを±1/±2循環（利得最大） |
| 風 | d8 | 8 | 旋回: 半回転（利得最大） |
| 水 | d20 | 20 | 流転: 他ダイスを振り直し（期待利得最大） |
| 闇 | d10 | 10 | 覚醒: 他ダイスを裏返し（利得最大） |
| 光 | d12 | 12 | 昇華: 他ダイスを最大値に（利得最大） |

### 3-3. 達成値

- **澱み**: d20は出目3以下、その他は出目1 → 達成値 **−1**
- **水(d20) 非澱み**: ⌊出目 ÷ 2⌋
- **その他**: 出目そのまま
- 出目は**循環**（Nの次は1）

### 3-4. 難易度帯（暫定目安）

| 達成値 | 目安 |
| --- | --- |
| 5–6 | 易しい |
| 7–8 | 普通 |
| 9–10 | 難しい |
| 11–12 | 困難 |
| 13–14 | 非常に困難 |
| 15–16 | 無謀 |
| 17+ | 不可能級 |

### 3-5. 限界貢献度（バランス指標）

```
marginal(d) = avg(プール ∪ {d}) − avg(プール)
```

属性ダイスの強弱・マナコスト設計の主指標。シム Part2 出力: `marginal_by_n.csv`

## 4. 種族（確定）

詳細: `legacy/races.md`

| 種族 | 判定時能力 | 要点 |
| --- | --- | --- |
| 人間 | 翻転 | 未反応1個を裏返し。コスト: 最終達成値 − ダイス総数 |
| 獣裔 | 転機 | 振り直し（下振れ→1固定）。未反応に戻り連鎖可 |
| 機巧 | 素読 | 判定前に d10→d12 換装。澱みなし |

各種族の**戦闘時能力**も確定（`legacy/races.md` §2 参照）。

## 5. 錬金・品質（確定）

詳細: `legacy/alchemy_and_items.md`

### 5-1. 錬金判定

- **プール**: 1d10 + 支払ったマナに対応する属性ダイス
- **難易度**: レシピ難易度 + 余剰タグ1つにつき +1
- **錬金値**: 判定の達成値
- **品質**: 熟練度 + 錬金値
- **想成使用時の品質**: 保存されている熟練度 + 保存されている錬金値

### 5-2. マナ所持上限（確定）

| 属性 | 上限 |
| --- | --- |
| 火・地・風・水 | 各5 |
| 闇・光 | 各1 |

### 5-3. アイテムの基本単位

- PCは**アイテム**を保持する。タグ単体は保持しない
- タグ `[名前]`: 分類（マナ対応）。印記 `<名前>`: 効果分類（`legacy/marks_and_tags.md`）

## 6. 戦闘（確定）

詳細: `legacy/combat.md`

### 6-1. 骨格

- **タイムライン**: 0〜11 の12カウント。ラウンド制
- **手札**: 3枚。手番ごとに1枚必須プレイ
- **行動段階**（カード数字）: A〜3=軽 / 4〜7=中 / 8〜10=重 / J〜K=特
- **基本待機値**: 軽3 / 中4 / 重5 / 特6
- **待機値**: 基本待機値 + 重量補正（軽−1 / 中±0 / 重+1）
- 探索中の戦闘は常に**簡易戦闘**（デフォルト2ラウンド上限）

### 6-2. 装備品の算出式（確定）

```
武器ダメージ = 品質 ÷ 等級 × 攻撃係数（切捨て）
防御値     = 品質 ÷ 等級（切捨て）
<装甲>強度 = 品質 ÷ 等級 × 防御係数（切捨て）
```

攻撃係数・防御係数・待機値は、**行動段階 × 装備重量**のテーブルで決まる（`legacy/alchemy_and_items.md` §10）。

### 6-3. 武器のカウントあたり効率（DPC）

各重量で最適行動段階が異なるよう、攻撃係数÷待機値が概ね均等になるよう設計済み。

| 重量 | 最適行動段階 | DPC目安 |
| --- | --- | --- |
| 軽量 | 軽行動 | ≈1.50 |
| 中量 | 中行動 | ≈1.50 |
| 重量 | 特行動 | ≈1.57 |

### 6-4. 消耗品の使用・想成（確定）

- **使用**: 所持アイテムを消費して効果発動。待機値 = 行動段階 + 重量補正
- **想成**: レシピ参照・マナ支払いで効果発動。スート一致マナ1つ無料可
- **攻撃の定義**: 武器または素手のみ。アイテムによるダメージは攻撃ではない
- **効果量の算出式**: ルール本文に**未記載**（→ §8 方針参照）

## 8. 合意済み設計方針（ルール未確定）

以下はデータ設計の方向として合意済み。`docs/` への反映・数値確定はこれから。

### 8-1. データ設計の出発点

現時点で**数値的に確定しているデータ**は以下のみとみなす。

1. ダイスシステム（反応・達成値・限界貢献度）
2. 六種族の判定時能力（Part4 で比較済み）
3. **§7 マナ1 ≈ +5** の換算基準

それ以外（タグ・アイテム・敵・HP・レシピ等）はルール骨格のみあり、**一覧データは未整備**。

### 8-2. 等級

| 項目 | 方針 |
| --- | --- |
| 想定段階 | おおむね **1〜5** の5段階 |
| 武器・防具 | 等級あり。**品質÷等級**で数値の暴れを抑え、見た目のレア度にも使う |
| 消耗品 | 等級を入れるか**未定**。マナ投入がすでに効果量に直結するため、二重管理を避けたい |

### 8-3. 消耗品の効果量

| 項目 | 方針 |
| --- | --- |
| 基本形 | `k × 品質`（等級なし）を第一候補 |
| 行動段階 | 消耗品の効果量を行動段階で変えない案を検討中 |
| 重量 | アイテム重量ごとに**効果量を一律**（行動段階に依存しない） |
| 投資との関係 | 錬金に費やしたマナが多いほど品質↑ → 1マナあたりの強さが上がる |

### 8-4. 武器と消耗品の使用タイミング

手札管理の戦略として、**武器重量とアイテム使用タイミングを逆相関**させる。

| 武器重量 | 武器に使うカード | 余りやすいカード | アイテムの使いどころ |
| --- | --- | --- | --- |
| 軽量 | 軽行動（A〜3） | 特行動（J〜K） | 高数字カードで強いアイテム |
| 中量 | 中行動（4〜7） | 軽・重・特 | 状況次第 |
| 重量 | 特行動（J〜K） | 軽行動（A〜3） | 低数字カードで軽いアイテム |

消耗品の効果を行動段階でスケールさせない場合、上記は**手札の「余り方」**として成立する（効果の強弱は錬金投資で決まる）。

### 8-5. HP

- **キャラ作成時の初期HP 100** は `legacy/character_creation.md` に記載があるが、**バランス設計の固定基準ではない**
- 敵HP・PC HP は、品質スケールと戦闘想定（1戦あたりのアイテム消費等）から**後続で導出**する

## 9. データ作成の優先順位

確定基準から派生する作成順の目安。

```
1. マナ換算基準（§7）          ← 確定
2. 消耗品効果式・k値           ← §8.3 で方針確定後
3. 等級テーブル（武器・防具）   ← §8.2
4. レシピ難易度（目標品質から逆算、Part3参照）
5. マナコスト（限界貢献度から算出、Part2参照）
6. タグ→マナ変換レート
7. 敵HP・攻撃力（戦闘想定から導出）
8. ドロップ・素材データ
```
