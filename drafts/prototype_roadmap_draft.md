# プロトタイプ・テストプレイ ロードマップ 草案

> 作業仮決定版（2026-04-11）  
> 目的：なるべく早くテストプレイを回せるための最小構成の整理。  
> 現状のアイデアは実装しつつ、感触を確かめることを最優先とする。

---

## テストプレイで確認したいこと

- タイムライン戦闘の行動順操作が**楽しいかどうか**
- 4クラス（爆弾 / 薬 / 香料 / 札）の**担当が実際に分かれて機能するか**
- アイテムを作って使うという**錬金術TRPGのループが成立するか**
- 1セッションの**密度・時間・テンポ**の感触

---

## テストプレイまでのステップ

### ステップ1：キャラクターの基本ステータスを決める

- [x] 初期行動値：戦闘開始時に `1d10` を振り `floor(n/2)` で決定（結果は0〜5、0と5が出にくい分布）
- [x] HP基準：達成値の基準（1d10 + マナ2〜3個分）の3〜4倍程度
- [x] HPの具体数値を仮決めする → **100**
- [ ] 防御値の扱いを決める（固定値か、ダイスか）

> キャラクターシートのレイアウトはテストプレイ後でよい。所持スロットは一旦考えない。

### ステップ2：テスト用レシピを作る（4系統 × 数種ずつ）

- [ ] 爆弾：ダメージ量・範囲・素材コストを決める（3〜5種類）
- [ ] 薬：回復量・治療効果・素材コストを決める（3〜5種類）
- [ ] 香料：強化内容（何をどれだけ上げるか）・素材コストを決める（3〜5種類）
- [ ] 札：デバフ・妨害・封印の具体効果・素材コストを決める（3〜5種類）

### ステップ3：戦闘ルールの最小版を固める

- [ ] HP・防御値の計算式またはデフォルト値を決める
- [ ] アイテムを使う際の手順（手番消費・対象指定）を整理する
- [ ] 「スタン」「デバフ」などの状態異常の処理を最小限で定義する
- [ ] 戦闘終了条件を決める（敵全滅のみで十分か）

### ステップ4：テスト用素材と調達ルートを用意する

- [ ] テスト用素材の種類をざっくり決める（5〜10種程度）
- [ ] 探索フェーズ抜きで素材を配布するだけのスタート方式でよいか検討する

### ステップ5：テスト用シナリオ（1戦闘）を作る

- [ ] テスト用敵を1〜2体用意する（HP・待機値・行動）
- [ ] 戦場のエリア構成を決める（シンプルに1〜2エリア想定）
- [ ] 戦闘前の準備フェーズ（アイテム選択）の流れを確認する
- [ ] 1セッションの流れ（準備 → 戦闘 → 振り返り）を書き出す

## 現時点の決定事項

### キャラクター作成

| 項目             | 内容                                                                |
| ---------------- | ------------------------------------------------------------------- |
| 系統選択         | テストプレイは4クラス固定（1系統のみ）                              |
| 系統一覧         | 爆弾 / 薬 / 香料 / 札                                               |
| 未習得系統       | レシピ使用不可                                                      |
| 町での顔         | 自由設定（機械的制約なし）                                          |
| 初期行動値       | 戦闘開始時に `1d10` を振り `floor(n/2)` で決定（0〜5）              |
| HP（仮）         | **100**（達成値基準の約5倍、平均4〜5撃耐久）                        |
| 初期敵火力（仮） | **25**（p50=17〜22 を上回る設定。無対策だと回復が間に合わなくなる） |
| 所持スロット     | 一旦考えない                                                        |
| 1日の製作数      | 既存ルールの町行動キャップに準拠（昼2回＋夕2回、最大4回）           |

### 消耗品

| 系統 | 担当                   | 特記                       |
| ---- | ---------------------- | -------------------------- |
| 爆弾 | 単発ダメージ           | デバフは副次効果まで       |
| 薬   | 単発回復・治療         | 他者へ使うには補助具が必要 |
| 香料 | 単発強化・バフ         | 他者へ使うには補助具が必要 |
| 札   | 単発デバフ・妨害・封印 | -                          |

### 戦闘

| 項目         | 内容                               |
| ------------ | ---------------------------------- |
| 進行方式     | タイムライン方式                   |
| 1ラウンド    | 12カウント（秒数は厳密に扱わない） |
| 軽行動       | 待機値 3                           |
| 中行動       | 待機値 4                           |
| 重行動       | 待機値 5                           |
| 特行動       | 待機値 6                           |

| 重量武器補正 | 全待機値 +1                        |
| 位置管理     | エリア制（グリッドなし）           |
| 同カウント   | PC側が先行（基本ルール）           |

## テストプレイ時点で除外するもの（後回し）

以下は設計済み or アイデアあり、だがテストプレイには不要なため保留。

| 除外項目                       | 理由                                 |
| ------------------------------ | ------------------------------------ |
| 銃・弾薬                       | 独自のパーツ差し替えメカニズムが重い |
| ポーション                     | タグ依存の特殊メカニズムが重い       |
| 装備製作（武器・防具）         | 打ち直し・強化メカニズムが重い       |
| 継続効果系（加護・呪具・兵器） | 単発の強さが決まってから設計         |
| 触媒                           | 副資材のため単独で機能しない         |
| 工房成長・系統習得             | ループ全体が確認できてから           |
| 補助具（スロワー等）           | 対象ルールは一旦「自分のみ」で運用   |
| 料理                           | 戦闘外専用のため戦闘テストには不要   |
| 探索フェーズ詳細               | 素材配布だけの簡易版で代用           |
