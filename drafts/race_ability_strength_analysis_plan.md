# 六種族・判定時効果 強さ比較 分析計画

作成: 2026-04-01  
目的: 六種族それぞれの判定時効果が達成値に与える影響をモンテカルロ法で定量化し、バランス調整の根拠データを得る

---

## 1. 分析の目的

属性ダイスの限界貢献度分析（`monte_carlo.c` Part2）と同じ手法で、  
**「種族効果を持っていない場合」と「種族効果を使った場合」を比較**し、各効果の強さを数値化する。

これにより以下が判断できる：

- 各種族の効果が強すぎる / 弱すぎるかの客観的根拠
- 付喪・半霊の2候補案（出目+1 vs 入れ替え）のどちらが適切か
- ホムンクルスの三択の実質的な強さ（三択全体 vs 最強択）

---

## 2. 測定指標

各種族効果について以下を計算する。

| 指標                | 説明                                                  |
| ------------------- | ----------------------------------------------------- |
| **期待値増分**      | 種族効果あり平均達成値 − 効果なし平均達成値           |
| **成功率テーブル**  | 目標値 5〜17 それぞれの成功確率（効果あり・なし両方） |
| **p50 / p90 / p99** | 中央値・上位10%・上位1%の達成値                       |

「限界貢献度」的に捉えるなら、**期待値増分**が主指標。  
成功率テーブルは難易度帯ごとの実感を確認するために補助的に出力する。

---

## 3. テストするプール構成

Part2の限界貢献度分析と同じく、**有効な全ダイスプール構成を列挙して総当たり**で計測する。  
特定の代表構成に限定するのではなく、あらゆるプールでの効果の強さを平均することで、  
ゲーム全体を通した種族効果の実質的な強さを測る。

### 列挙範囲（Part2と同一条件）

| 条件                      | 内容                              |
| ------------------------- | --------------------------------- |
| **d10の個数**             | 1個以上（`counts[d10] >= 1`）必須 |
| **d4/d6/d8/d20 の各個数** | 0個以上、合計が `MAX_N` 以下      |
| **d12 の個数**            | 0個または1個（別枠）              |
| **合計ダイス数**          | 1〜`MAX_N`（現在 `MAX_N = 10`）   |

構成総数はPart2と同じ数万〜数十万オーダーになる。

### 集計方法

各構成 S に対して「種族効果あり平均達成値 − なし平均達成値」= `delta(S)` を計算し、  
全構成を通じた `E[delta]` を**種族効果の総合的な強さ（期待値増分）**として採用する。

Part2の「横軸=プール内の他ダイス合計個数」に相当する分解も行い、  
プールが小さい / 大きい場合で効果の強さがどう変わるかをグラフ化できるようにする。

| 指標                  | 説明                                        |
| --------------------- | ------------------------------------------- |
| **avg_delta（総合）** | 全構成を通じた期待値増分の平均              |
| **avg_delta_by_n[k]** | プール合計 k 個の構成のみでの期待値増分平均 |
| **sample_count**      | 集計に使った構成数（信頼性の確認用）        |

## 4. 各種族効果の実装方針

### 前提共通事項

- 種族効果を**最適なタイミング・最適な対象**に使う、という**最善手プレイヤー仮定**を採用する
- 「任意タイミング」の効果は、**全反応処理の完了後に使用**するケースを標準とする  
  （補足として「反応処理前に使用」「反応の途中で使用」のケースも後から検討可）
- 全効果は1判定につき**1回のみ**使用可とする

---

### 4-1. 人間：裏返し

**効果**  
全リアクション処理後、未反応/反応済みを問わず、ダイス1つを選んで裏返す（反応済みにする）。

**実装方針**

- 全ダイスの中で `invert_face(sides, face)` 後の利得が最大のダイスを選ぶ
- 利得 ≤ 0 なら使用しない（任意発動）

**特記事項**

- d20 は裏返すと出目が大きく変わる可能性があるため、変換後の達成値で利得を計算する必要がある
  （d20 は `floor(face/2)` 変換があるため、face 単純反転ではなく達成値で比較すること）

---

### 4-2. 機械人：d12 置換

**効果**  
振る前に d10 → d12 に置換してよい。そうした場合、当該判定でダイス反応は一切行えない。  
また置換した d12 の出目1は 1 として扱う（-1 にならない）。

**実装方針**  
以下の2ケースをそれぞれシミュレートして比較する：

- **Aケース**: d12 置換あり（反応なし、出目1=1）
- **Bケース**: d10 そのまま使用（通常通り反応あり）

モンテカルロ上は両方を計算し、高い方の期待値を採用した場合の平均値を「機械人効果あり」とする。

**補足**  
実際のゲームプレイでは、プレイヤーが事前に判断するため、  
「期待値で上回る構成のときのみ置換を選ぶ、ということを前提にした平均値」が適切。

---

### 4-3. 獣人：ペナルティ付き振り直し

**効果**  
全リアクション処理後、ダイス1つを選んで振り直す（反応済みにする）。  
振り直し後 < 振り直し前 の出目なら、そのダイスは強制 -1（ただし他の効果で出目が変わると解除）。

**実装方針**

- 振り直し後の期待値を現在値と比較し、期待値 > 現在値 のダイスを選ぶ
- 期待値 ≤ 現在値 なら使用しない（任意発動）
- ペナルティ判定: 振り直し後の出目 < 振り直し前の出目なら達成値計算で -1 に固定
- 「ペナルティ解除（他効果による変更）」はこの単独分析では発生しない前提とする

**計算上の注意**  
期待利得の計算には以下が必要：

```
E[振り直し後の達成値] = Σ_{f=1}^{N} (1/N) × (出目 f 時の達成値ただしf<現在値なら-1)
```

これは「通常の期待値」よりも低く出る設計になっているはず。

---

### 4-4. ホムンクルス：後出し三択

**効果（いずれか1つ）**  
A: ダイス1つの -1 を 1 として扱う  
B: ダイス1つを達成値計算から除外し、別のダイス1つを最大値として扱う  
C: ダイス2つを選び、片方の出目をもう片方と同じにする

**実装方針**

- 全反応処理後、A・B・C それぞれを全ダイスの組み合わせで試し、最も達成値が上がる選択を採用する
- A は出目が-1のダイスが存在する場合のみ有効
- B は「除外の損失 < 最大値設定の利得」の場合のみ有効
- C は2ダイス選択全パターンを列挙し（方向を含め）最大利得を選ぶ
- いずれの択も利得 ≤ 0 なら使用しない（任意発動）

**補足**  
三択全体の強さだけでなく、「A / B / C それぞれが選ばれた割合」も記録すると、三択の実質バランスが見えて調整に役立つ。

---

### 4-5. 付喪候補 A：未反応ダイスの出目 +1

**効果（候補）**  
未反応ダイス1つを選び、出目を1進める（循環）。そのダイスは反応済みにする。

**実装方針**

- 未反応ダイスが対象（反応済みは除外）
- 全反応処理後に使用（タイミングは要検討）
- `cyclic_adjust(sides, face, +1)` 後の利得が最大のダイスを選ぶ
- 利得 ≤ 0 なら使用しない

---

### 4-6. 付喪候補 B / 半霊候補 A：ダイス2つの出目を入れ替える

**効果（候補）**  
ダイス2つを選び、それぞれの出目を入れ替える（反応済み状態は変化しない）。

**実装方針**

- 全ダイスペアの全順列を列挙し、入れ替え後の達成値合計が最大になるペアを選ぶ
- 入れ替えによる利得 ≤ 0 なら使用しない

---

### 4-7. 付喪・半霊 各候補の割り当てパターン

付喪・半霊それぞれに以下の2候補をシミュレートし、4通りの組を比較する。

|          | 候補A（出目+1） | 候補B（入れ替え） |
| -------- | --------------- | ----------------- |
| **付喪** | 付喪A           | 付喪B             |
| **半霊** | 半霊A           | 半霊B             |

→ 最終的に強さが似通っているほうを似た種族に割り当て、差別化できる組み合わせを探す。

## 5. Cコードへの追加方針

`monte_carlo.c` に **Part 4** として関数を追加する。  
構造はPart2の全列挙ループに準じ、`run_full_enumeration()` を改造・拡張する形を取る。

### 追加する関数

```c
// 種族効果なしのベースライン達成値を返す（反応処理完了後の状態を受け取る）
static int score_baseline(const Die *dice, int n_dice);

// 各種族効果を最適に適用し、適用後の達成値を返す
// （反応処理完了後のダイス配列のコピーを受け取り、元の配列は変更しない）
static int score_human       (Die *dice, int n_dice);  // 裏返し
static int score_beast       (Die *dice, int n_dice);  // ペナルティ付き振り直し
static int score_homunculus  (Die *dice, int n_dice, int *out_option_chosen);
static int score_tsukumogami_A(Die *dice, int n_dice); // 未反応+1
static int score_tsukumogami_B(Die *dice, int n_dice); // 入れ替え

// 機械人の「d12置換あり」版プールを1試行ぶんシミュレートして達成値を返す
static int score_machine_d12(const int *pool_sides, int n_pool,
                              WindMode wind_mode);

// 全構成列挙・種族効果強さ比較
// Part2 の列挙ループと同じ構成範囲をすべて走査する
static void run_race_ability_comparison(int trials_per_pool, WindMode wind_mode,
                                        const char *out_dir);
```

### 計測フロー（構成 S を固定した場合・1試行）

```
1. 構成 S のダイスを全部ロールし、全反応処理を完了させる
2. 反応処理後のダイス配列をコピーして保存（baseline_dice[]）
3. score_baseline(baseline_dice, n) → baseline_score
4. 各種族効果適用関数に baseline_dice のコピーを渡す
   → ability_score[種族]
5. delta[種族] = ability_score[種族] - baseline_score を記録
```

`trials_per_pool` 回繰り返して、構成 S における `E[delta[種族]]` を得る。  
これを全構成にわたって集計することで `E_all[delta[種族]]` を得る。

### 集計配列の構造

```c
#define N_RACES 7  // 0:人間, 1:機械人, 2:獣人, 3:ホムンクルス,
                   // 4:付喪A, 5:付喪B(入替), 6:半霊(付喪と同候補を逆に割当)

double delta_sum_all[N_RACES];         // 全構成の合計
long   delta_cnt_all[N_RACES];
double delta_sum_by_n[N_RACES][MAX_N]; // プール合計個数 n 別の合計
long   delta_cnt_by_n[N_RACES][MAX_N];

// ホムンクルス専用: 三択の選択率
long   homo_option_cnt[3];  // A選択 / B選択 / C選択
```

**機械人の特例処理**  
機械人の「d12置換」は「振る前」に決断するため、1試行を2回走らせて比較する。

- ケース1: 通常d10プールを普通に走らせて `score_normal`
- ケース2: d10をd12に置換し反応なしで走らせて `score_machine_d12`
- `max(score_normal, score_machine_d12) - score_normal` を delta として記録  
  （プレイヤーが毎回期待値で最善側を選ぶ仮定）

## 9. 付喪・半霊の判定基準

分析後、以下の観点で付喪・半霊の効果を選定する：

- **期待値増分が人間・獣人と大きく乖離していないか**（極端に強すぎ/弱すぎを排除）
- **付喪と半霊の強さが差別化されているか**（同じ候補を両方に割り当てるのは避ける）
- **テーマとの整合性**：付喪は「依代との結びつき・安定」、半霊は「境界の曖昧さ・ずらし」

目安として、**期待値増分 ±0.5 以内に全6種族が収まる**ことを最初の目標とする。  
（現行ダイス分析での d4〜d20 の限界貢献度の幅を参考に調整）
