## 仮採用内容（検討対象）

### A. ラウンドの定義 ✅ 確定

- 1ラウンド = **12カウント**
- カウントは **0〜11 の12段階で循環**する（カウント11の次は0に戻る）
- ラウンドは補助単位。カウントだけが実際に進行する単位であり、ラウンドはカウントへの別名ラベルでしかない
- 実時間の秒数は厳密には扱わない。演出的には1分弱程度で見てよい
- 簡易戦闘はカウント0〜11（1ラウンド）または0〜23（2ラウンド）しか進行しない

> **→ 現行 combat.md の「カウント1の直前の行動が終わった時点を1周」という記述は、この定義に合わせて更新が必要。**

| -------- | ---------- |
| 軽行動   | 3          |
| 中行動   | 4          |
| 重行動   | 5          |
| 特行動   | 6          |

### C. 重量補正

| 武器重量 | 全待機値への補正 |
| -------- | ---------------- |
| 軽量武器 | −1               |
| 標準武器 | 補正なし         |
| 重量武器 | +1               |

重量補正は武器個性の全部ではなく、**速さの位置づけ**に使う。  
武器個性の大部分は「得意段階ボーナス」で表現する方針。

**待機値の下限**: ✅ 下限なし（軽行動＋軽量 = 2カウント を許容）  
→ 12カウントラウンドで最速6回行動になるが、これは意図的に許容する。

### D. カード数字の扱い ✅ 確定

- カードの数字をそのまま待機値として使う案は廃止。
- 代わりに、数字を**4帯**に区切り、帯が**武器固有の追加ボーナスのトリガー**になる。
- 帯は行動の制限ではない。どの帯のカードでも任意の行動段階の攻撃を宣言できる。

#### カード帯区分

| 帯       | 数字範囲 | 枚数（スート5種）4枚分 | 対応する行動イメージ |
| -------- | -------- | ---------------------- | -------------------- |
| 軽行動帯 | A～3     | 3枚×4スート = 12枚     | 軽い行動に対応       |
| 中行動帯 | 4～7     | 4枚×4スート = 16枚     | 中程度の行動に対応   |
| 重行動帯 | 8～10    | 3枚×4スート = 12枚     | 重い行動に対応       |
| 特行動帯 | J～K     | 3枚×4スート = 12枚     | 特殊・大技に対応     |

> 絵札（J/Q/K）がそのまま特行動帯になる。直感的な割り当て。  
> 特行動帯の枚数がやや少ない（23%）のは意図的。切り札級の重みが生まれる。

#### 帯の機能

- 帯は**補正型**。どの帯のカードでも任意の行動段階の攻撃を宣言できる
- 宣言した**行動段階**と出した**カード帯**の組み合わせで、武器固有のボーナスが発動する場合がある
- 「重行動を宣言し、軽行動帯カードを切る」のも選択可能。ただし武器ボーナスは帯条件を満たすか否かで変わる
- **全武器共通のカード帯ボーナスはなし**。帯ボーナスは武器側で個別に定義する

**未決論点 D-1**: ~~帯ごとの共通ボーナス~~ → 廃止。各武器の追加ボーナスとして武器側で設計する

### E. 係数とボーナスの定義 ✅ 仮値確定

#### ダメージ公式

```
ダメージ = base × 行動段階係数
base = floor(品質 ÷ 4)
```

- **品質**: 武器固有の数値。アイテム・装備共通スケールで管理する。
- **base**: 品質を4分の1にした基礎値（端数切り捨て）。係数・ボーナスの基準単位。
- **行動段階係数**: 待機値と重量クラスで決まる整数。個別武器ではなく**重量3種テーブル**で管理。

---

#### 行動段階係数テーブル ✅ 仮値確定

> **設計原則**:
>
> - 軽量武器: 軽い行動ほど効率が高い。全体的にDPSが高めで防御値の影響を受けやすい。
> - 標準武器: 中行動・重行動付近が最も効率的。中間的な設計。
> - 重量武器: 重い行動ほど効率が高い。D=0では最弱だが防御値が高い敵に最も強い。

**待機値（重量補正込み）:**

| 行動段階 | 軽量wait | 標準wait | 重量wait |
| -------- | -------- | -------- | -------- |
| 軽行動   | 2        | 3        | 4        |
| 中行動   | 3        | 4        | 5        |
| 重行動   | 4        | 5        | 6        |
| 特行動   | 5        | 6        | 7        |

**係数:**

| 行動段階 | 軽量係数 | 標準係数 | 重量係数 |
| -------- | -------- | -------- | -------- |
| 軽行動   | **3**    | 3        | 3        |
| 中行動   | **4**    | **5**    | 4        |
| 重行動   | **5**    | **6**    | **6**    |
| 特行動   | **5**    | **6**    | **9**    |

**各行動段階のDPS（÷待機値、×base）:**

| 行動段階 | 軽量 DPS   | 標準 DPS   | 重量 DPS   |
| -------- | ---------- | ---------- | ---------- |
| 軽行動   | **1.50** ★ | 1.00       | 0.75       |
| 中行動   | **1.33** ★ | **1.25** ★ | 0.80       |
| 重行動   | 1.25       | **1.20** ★ | 1.00       |
| 特行動   | 1.00       | 1.00       | **1.29** ★ |

> 軽量: 軽行動が最も効率よく、行動が重くなるほど効率が落ちる。
> 標準: 中行動・重行動がほぼ同率で最も効率的。
> 重量: 特行動が突出して最強。軽い行動は積極的に非効率。

**防御値別の平均DPS（base=6、全段階均等平均）:**

| 防御値 | 軽量    | 標準    | 重量    | 最強 |
| ------ | ------- | ------- | ------- | ---- |
| D=0    | **7.6** | 6.7     | 5.8     | 軽量 |
| D=5    | **6.0** | 5.5     | 4.8     | 軽量 |
| D=10   | **4.4** | 4.3     | 3.9     | 軽量 |
| D=15   | 2.8     | **3.1** | 2.9     | 標準 |
| D=20   | 1.2     | 1.9     | **2.0** | 重量 |
| D=25   | 0       | 0.7     | **1.0** | 重量 |

> クロスオーバー: 軽量→標準 D≈**11**、標準→重量 D≈**19**
> D=0〜10で軽量最強、D=10〜20で標準最強、D=20以上で重量最強。

#### HP100・3〜5発検証

品質 N=3〜5ダイス期待値 ≈ 22〜34、base = floor(品質÷4) ≈ 5〜8。  
最も一般的な攻撃（標準武器・中行動）:

```
ダメージ = base × 5 = 6 × 5 = 30  →  100÷30 ≈ 3.3発  ✓
```

### F. 武器設計方針 ✅ 確定

**PLへの提示方針**:

- 武器は**重量クラス**（軽量・標準・重量）と攻撃印記のみ提示する
- 斬撃/打撃/刺突の系統分類表はPLには出さない
- 各武器には攻撃印記（`<斬撃>` `<打撃>` `<刺突>`）を付与し、戦術判断に使えるようにする
- 複合印記も許容する（例: `<斬撃><打撃>`）

**設計側の内部指針**（PLには非提示）:

- `<斬撃>` → ダメージ重視
- `<打撃>` → ノックバック・スタン重視
- `<刺突>` → デバフ・制御重視
- この三分法は**新武器追加時のガードレール**として使い、効果が偏って重複しないよう管理する
