## 1. 戦闘の基本方針

戦闘は、探索と接続された処理として扱う。  
簡易戦闘と通常戦闘は、できるだけ同じ骨格で回す。

### タグと印記の詳細

- タグと印記の共通定義・記述方針・参照ルールは `marks_and_tags.md` を参照する

基本思想は次の通り。

- 探索戦闘は短く、軽く、判断重視
- 通常戦闘は同じ骨格の拡張版として重くする
- プレイヤー側の操作はできるだけ共通化する
- 探索では災厄だった Joker が、戦闘では切り札になる
- ダメージだけでなく、**行動順の操作**を重要な要素にする

## 2. 進行方式

- 戦闘は**タイムライン方式**で進行する
- 各キャラクターと敵は、タイムライン上にコマを持つ
- カウントが最も小さい者から順に行動する

### ラウンド

- 戦闘は厳密なラウンド制ではなく、タイムライン上で継続的に進行する
- ただし、時間経過や制限を扱うためにラウンドという補助単位を置く
- 1ラウンド = **12カウント**。カウントは **0〜11 の12段階で循環**し、カウント11の次は0に戻る
- ラウンドは補助単位であり、実際に進行する単位はカウントのみ
- 簡易戦闘はカウント0〜11（1ラウンド）または0〜23（2ラウンド）のみ進行する

### 同カウント

- 同じ陣営内の順番は、その陣営の担当者が決める
- 敵味方が同じカウントの場合、基本はPC側が先に行動する
- 先制、奇襲、ボス特性などで例外を設定してよい

## 3. 戦場の考え方

- 位置関係はマス目ではなく**エリア**で管理する
- 対象指定は原則として **単体 / 1エリア / 全体** に統一する

### エリア数

- 簡易戦闘: 少数エリアで処理する
- 通常戦闘: エリア数を増やして位置取りを扱う

### 移動

- 複数エリアがある戦闘では、移動は**隣接エリア間の移動**として扱う

## 4. 山札と手札

### 山札

- 戦闘では、その探索中に使っているトランプの山札をそのまま使用する
- 探索で既に使われた札は、戦闘中には出ない
- 山札が尽きた場合は、捨て札をシャッフルして新しい山札を作る

### 手札

- 戦闘開始時、各PCとGMは**手札を3枚引く**
- 戦闘終了時、手札はすべて**捨て札に送る**

### 手番での使用

- 手番が来たら、行動に合わせて**必ず手札を1枚プレイする**
- カードを使わずに行動することはできない

### 補充

- 手札が**0枚の状態で自分の手番が来た時**に、山札から新たに3枚補充する

### 敵手札

- 敵の手札は、基本的に**完全公開**とする
- 敵手札は、**GM側の共有手札**として扱う
- 共有敵手札が0枚なら、次に行動する敵の手番開始時に3枚補充する
- 非公開は例外的な特殊状況のみでよい

## 5. 手番の流れ

各ユニットの手番では、基本的に以下を行う。

1. 行動を宣言する
2. その行動に対応する手札を1枚プレイする
3. 必要なら対象を指定する
4. カードの数字とスートを適用する
5. コストを消費する
6. 必要なら打ち消し判定を行う
7. 行動を解決する
8. そのカードに応じた待機値ぶん、タイムライン上で後方へ進む

### 基本行動

- **攻撃**: 武器または素手を用いて対象にダメージを与える行動のみを「攻撃」とする。レシピ使用・アイテム使用はたとえダメージを与えても「攻撃」ではない。
- スキル使用
- アイテム使用
- 防御
- 回避
- 分解
- 離脱
- 戦場オブジェクトへの干渉

簡易戦闘では移動を使わない。  
また、**すべての行動は主行動を消費**する。

### 打ち消しと消費

- 打ち消し判定は、**コスト/手札消費直後、効果解決前**に行う
- 対象を取る行動は、対象指定までは行い、その後に不発になってよい
- アイテムは使用宣言時点で即消費し、打ち消されても戻らない
- スキルも宣言時点でコスト消費を確定し、打ち消されても戻らない
- 行動が打ち消されても、プレイした手札は消費される

## 6. カードの意味

### 数字とカード帯

- カードの数字は行動待機値に**直接は使わない**
- 数字は**4つの帯**に区切られ、帯が武器・アイテムの追加ボーナスのトリガーになる
- どの帯のカードでも任意の行動段階の攻撃を宣言できる。帯は行動の制限ではない

| 帯       | 数字範囲 | トリガー印記 |
| -------- | -------- | ------------ |
| 軽行動帯 | A〜3     | `<軌転>`     |
| 中行動帯 | 4〜7     | `<定常>`     |
| 重行動帯 | 8〜10    | `<励起>`     |
| 特行動帯 | J〜K     | `<臨界>`     |

- 宣言した行動段階と出したカード帯の組み合わせで、武器固有のボーナスが発動する場合がある
- **全武器共通のカード帯ボーナスはなし**。帯ボーナスは武器側で個別に定義する
- 各トリガー印記の発動条件詳細は `marks_and_tags.md` を参照する

### スート

- スートは、**対応する戦闘用マナの系統**として扱う方向で進める
- 行動が対応するマナを使う行動であるなら、そのマナ1つを無料にする

### スートボーナスの原則

- スートが一致しなくても行動は可能
- 一致していると追加恩恵がある
- 現時点では、追加恩恵は**マナ1無料**のみを採用する
- 無料化分を持ち越したり蓄積したりはしない

## 7. 行動段階と待機値

攻撃などの行動は4段階に区分される。各段階には固有の**共通待機値**がある。

### 行動段階と共通待機値

| 行動段階 | 共通待機値 |
| -------- | ---------- |
| 軽行動   | 3          |
| 中行動   | 4          |
| 重行動   | 5          |
| 特行動   | 6          |

### 武器重量による補正

| 武器重量 | 全待機値への補正 |
| -------- | ---------------- |
| 軽量武器 | −1               |
| 標準武器 | 補正なし         |
| 重量武器 | +1               |

- 重量補正は武器の**速さの位置づけ**として使う
- 武器個性の大部分は帯ボーナスで表現する方針
- 待機値の下限はなし（軽行動＋軽量 = 2カウント を許容）

## 8. ダメージ公式と係数テーブル

### ダメージ公式

- `ダメージ = base × 行動段階係数`
- `base = floor(品質 ÷ 4)`

用語:

- **品質**: 武器・装備固有の数値。アイテム・装備共通スケールで管理する
- **base**: 品質を4分の1にした基礎値（端数切り捨て）
- **行動段階係数**: 行動段階と重量クラスで決まる整数。以下テーブルで管理する

### 行動段階係数テーブル（仮値）

| 行動段階 | 軽量係数 | 標準係数 | 重量係数 |
| -------- | -------- | -------- | -------- |
| 軽行動   | 3        | 3        | 3        |
| 中行動   | 4        | **5**    | 4        |
| 重行動   | 5        | **6**    | **6**    |
| 特行動   | 5        | **6**    | **9**    |

> 係数値は仮値。実戦テスト後に調整する。

### 追加ボーナス

- 係数だけでは表現できない武器個性を、必要な武器種にだけ追加で持たせる
- 追加ボーナスはトリガー印記（`<軌転>/<定常>/<励起>/<臨界>`）を使い、`<印記名>：〔効果〕` の形式で武器データに記載する
- 「無個性な武器」（剣など）は係数テーブルのみ。追加ボーナスなし

## 9. 属性耐性・弱点・ノックバック・スタン

### ダメージ処理順

1. 命中低下デバフ判定
2. 回避判定
3. 属性耐性適用（反射 / 無効 / 半減 / 通常）
4. 防御軽減量を減算
5. 最終ダメージ確定
6. 必要ならスタン付与判定

### 属性表

- 属性印記は `<火>` / `<地>` / `<風>` / `<水>` / `<闇>` / `<光>` / `<無属性>` を扱う
- 通常攻撃は `<無属性>` として耐性表を参照する
- `<無属性>` は戦闘上の属性区分であり、`alchemy_and_items.md` で定義されるマナ6属性とは別概念として扱う

### 1属性1状態

- 1つの属性には、**弱点 / 通常 / 半減 / 無効 / 反射** のどれか1つだけを設定する
- 弱点と耐性の同時指定は禁止する

### 優先順位

- 属性耐性の優先順位は **反射 > 無効 > 半減 > 通常** とする

### ノックバック

- **ノックバックX**は、自身のカウントをX進める（タイムライン後方へ送る）効果である
- カウントがまたタイムラインを周回する場合は、次ラウンドの対応するカウント位置に配置する

### 弱点

- 敵には弱点印記や弱点タグを設定できる
- 弱点を突いた場合は、通常のダメージ増加に加えて、**追加ノックバック**と**スタン付与判定**を行う
- `<無属性>` 弱点でも同様に扱う
- 複数対象攻撃では、弱点を突いた対象ごとに個別適用する

### 無効

- 無効時はダメージ0になる
- ただし、追加効果は発生する

### 反射

- 反射時は、**すべての計算前の元ダメージ値**を攻撃者へ返す
- 反射ダメージは防御/回避不可の確定ダメージである
- 反射した対象への元攻撃はその時点で解決終了し、追加効果も発生しない
- 複数対象攻撃では対象ごとに個別解決する

### スタン

- スタンは印記ではなく、**戦闘共通ルール**として管理する
- 各PCは「スタン値」と「スタン最大値」（基準=5）を持つ
- スタン値は戦闘中に蔓積し、**戦闘終了時に0へ戳る**
- スタン値がスタン最大値以上になったとき
  1. スタン値を0に戻す
  2. 手札からランダムに1枚捨てる
  3. 捨てた手札に応じてノックバック発生（詳細はノックバックの項を参照）

### ノックバックハメ

- ノックバックによる一方的な封殺は、**完全には禁止しない**
- 特に簡易戦闘では、それが成立するならそれも有効な勝ち方とする
- 通常戦闘・ボス戦では、耐性や部位差、複数行動体などで調整する

### 個別無効・個別反射

- 属性耐性表とは別に、敵能力や戦場ルールとして**個別無効・個別反射**を持たせてよい
- 例: デバフ無効、ポーション回復打ち消し、特定種別の効果反射
- 個別無効・個別反射は、通常の効果解決より**先に判定**する

### 命中低下の重複

- 命中低下は、同名は高い方で上書きする
- 将来的に別名の命中低下を導入する場合は、別名同士は共存し、**合計値で1回だけ判定**する

## 10. 離脱と撤退

### 離脱

- 各PCは、自分の手番に**タイムラインから自分のコマを取り除く**ことで離脱したものとして扱う
- 離脱は宣言時に即時成立する
- 離脱したPCは、その戦闘中に再参加できない

### 撤退成立

- PC全員のコマがタイムラインから取り除かれており、かつ**1人でもHPが0でないPCがいる**場合、その戦闘は撤退として扱う

### 離脱不能にしてよい場面

- Jokerイベント
- ボス戦
- GMが特殊状況として指定した場合

## 11. 戦闘不能

- HPが0以下になったPCは**戦闘不能**になる
- そのPCのコマはタイムラインから取り除く
- 戦闘不能が解除されるまで戦闘に参加できない
- 戦闘不能の解除は、味方の**アイテム使用またはスキル使用**で行う
- 解除されたPCは、**解除した味方の次手番開始時**に行動可能になる
- 戦闘不能になっても所持マナは失われない
- 戦闘不能になった敵にかかっていた効果は、その場で全消滅する

## 12. 防御・回避・分解

### 防御

- 防御行動は、次の自分の手番まで有効とする
- `1d10` と任意の追加マナダイスを振り、その合計を軽減量とする
- 追加ダイスが単一マナ種だった場合、その属性攻撃に対して半減を適用する
- 半減は端数切り捨て
- 防御で得た軽減量は、持続中に攻撃を受けるたび毎回フル値を適用する
- 防御系効果が複数ある場合は、同名は高い方で上書き、別名は共存して合計軽減する
- 同じ属性半減は重複しない
- 異なる属性半減は同時に持ってよい

### 回避

- 回避は独立した基本行動でもあり、また状態効果としても付与されうる
- 回避行動を取ると、次の自分の手番まで**回避値[行動枠] +4** を得る
- 回避値は **バフ枠** と **行動枠** の二枠で管理する
- バフ枠は上限5、行動枠は回避行動由来の固定4とする
- 被弾時に `1d10` を振り、出目が合計回避値以下なら回避成功とする
- 回避値が有効な間は、攻撃を受けるたび毎回判定する
- 自力では同一手番に防御と回避行動は併用できない
- ただし外部効果で併用状態になった場合、回避失敗時のみ防御軽減を適用する

### 分解

- 分解は、手持ちアイテム1つを消費して、そのアイテムに記載された**全タグ分**のマナを得る行動である
- 分解可否はタグではなく、**種別ルール**で管理する
- 装備は分解可能
- スコーリオンなど進行に関わる種別は分解不可
- 想成で作られたものは分解不可
- 装備を分解した場合、その装備効果は宣言時に即時消失する
- 分解で得たマナは、その手番の主行動では使えない
- 次の自分の手番から使用可能とする
- 共通割り込みルールは置かず、個別効果文で許可される場合のみ割り込みを認める

## 13. Joker

- Joker は戦闘中、**任意のスート・任意の数字**として扱える
- 使用時に、どのスート・どの数字として使うかを宣言する
- 現時点では、この機能のみを基本とし、専用追加効果は後で必要に応じて付け足す

## 14. 簡易戦闘

簡易戦闘は、探索中に発生する軽量な戦闘である。  
重視するのは殲滅よりも、**被害を抑えて切り抜けること**である。

### 特徴

- エリア数は少ない
- 敵は単純な構造
- 規定カウント到達で終了する
- 移動を使わない

### 勝利条件

- **PCたちが全滅せずに規定カウントへ到達したら勝利**
- 敵を全滅させる必要はない

### 終了処理

- 規定カウント到達時点で戦闘は即終了する
- 手札プレイによる行動は、効果解決後にカウント進行する
- その結果として規定カウントを超えた場合も、行動は解決済みとして扱う
- 使った手札は消費済みである
- 「次手番に発動」などの継続予約効果は、戦闘終了時に不発となる

### 報酬

- 報酬は、**倒した敵1体ごとに素材1つ**を基本案とする
- 規定カウントをやり過ごしても、敵を1体も倒していなければ報酬はない

### 打ち消しと公開情報

- 戦場ルールや敵能力による打ち消しは、システム設計上**事前に公開されるべき情報**として扱う
- ただし、卓運用としてどこまで厳格に守るかはGM裁量に委ねる
- 地景ルールは地景に入った時点で、敵能力は敵データの説明として提示されるのが望ましい
- アイテムの基本使用タイミングは `day_cycle.md` を正本とする
- 行動は、打ち消されると分かっていても宣言可能とする
- 打ち消し判定は、**コスト/手札消費直後、効果解決前**に行う
- 対象を取る行動は、対象指定までは行い、その後に不発になってよい
- アイテムは使用宣言時点で即消費し、打ち消されても戻らない
- スキルも宣言時点でコスト消費を確定し、打ち消されても戻らない
- 行動が打ち消されても、プレイした手札は消費される
- GMは「この行動は打ち消されるが、それでも行うか」を確認してあげるのが望ましい

## 15. 通常戦闘とボス戦

通常戦闘は、主依頼・ボス・攻略対象との重い戦闘である。  
簡易戦闘と同じ骨格を用いながら、より複雑な敵と戦場で重さを出す。

### 特徴

- エリア数が増える
- 敵が複数部位や複数対象を持ちうる
- 本体以外に装置・召喚源・結界などが存在しうる
- 弱点やタグ付与の順番が重要になる

### 通常敵との違いとして確定している点

- **撤退を選べない**
- **ターン数上限がない**

## 16. 敵行動パターン

敵の行動パターンは、カードの帯区分に対応する4種の行動を敵データとして定義する方式を採る。

### 敵サイズと待機値補正

- 敵は小型・中型・大型の3サイズに分類する
- 待機値補正はPC側の重量クラスと同じテーブルを使う（小型: −1 / 中型: 補正なし / 大型: +1）

### 行動テーブルの構造

- 各敵キャラクターは、軽行動帯・中行動帯・重行動帯・特行動帯それぞれの行動を定義する
- GMは共通の敵手札3枚を使い、引いたカードの帯から対応する行動を選んで実行する
- 効果量はHP・危険度などをもとに敵ごとに個別設定する

## 17. GM向け武器設計指針

- 武器をPLに提示する際は**重量クラス**（軽量・標準・重量）と攻撃印記のみ提示する
- `<斬撃>/<打撃>/<刺突>` の系統分類はPLには提示しない

### 系統ごとの設計方針

| 攻撃印記 | 設計方針                 |
| -------- | ------------------------ |
| `<斬撃>` | ダメージ重視             |
| `<打撃>` | ノックバック・スタン重視 |
| `<刺突>` | デバフ・制御重視         |

この三分法は新武器追加時の**ガードレール**として使い、効果が偏って重複しないよう管理する。
