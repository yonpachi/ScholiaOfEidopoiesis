## 1. 戦闘の位置づけ

戦闘は探索中に発生する処理である。  
選んだイベントが戦闘だった場合、その瞬間に戦闘が始まる。  
戦闘が終わったら、特記がない限り探索へ戻る。

探索中のイベントとして発生する戦闘は、常に**簡易戦闘**である。  
簡易戦闘以外の重い戦闘（ボス戦など）の細則は、必要になった時点で追記する。

## 2. タイムライン方式

戦闘はタイムライン上をカウントが進んでいく形で進行する。

- タイムラインは **0〜11 の12カウント**で構成される
- 0から始まり、1、2、3……と1カウントずつ進む
- 現在のカウントにコマが置かれているキャラクターは、順に手番を行う
- 手番を終えたキャラクターは、行動によって算出された**待機値**ぶん、コマを後方へ進める
- コマを後方へ進めた結果、11を超える場合、次のラウンドの対応するカウントにコマを置く
- 現在のカウントにあるすべてのコマが手番を終えたら、次のカウントに進む
- カウントが11から0に戻った時、ラウンドが1進む

### 同カウント内の処理

同カウントに複数のキャラがいる場合、**PCから**行動を処理する。  
PC同士の順番はPL側で任意に決めてよい。敵同士の順番はGMが決定する。

## 3. 戦闘の開始

戦闘が始まったら、以下の順に処理する。

1. 初期カウント決定
2. 手札の準備

### 3-1. 初期カウント決定

各PCと敵は、戦闘開始時にそれぞれ1d10を振る。  
その出目の半分（端数切り捨て）の値を、そのキャラクターの初期カウントとする。  
各キャラクターのコマを決定した初期カウントに配置する。

HPが0の状態で戦闘に入る場合、最初から戦闘不能として扱う。この場合、コマの配置と手札の準備は行わない。

### 3-2. 手札の準備

- 戦闘では、探索中に使っていたトランプの山札をそのまま使う
- 盤面に配置された札は、戦闘には使用しない
- 各PCとGMは、それぞれ山札から手札を**3枚**引く
- 山札が尽きた場合は、捨て札をシャッフルして新しい山札を作る

## 4. 手番でやること

自身の手番では、以下の順に処理する。

1. 行動宣言
2. カードプレイ
3. 行動段階決定
4. 行動解決
5. カウント変更
6. 手番終了

### 4-1. 行動宣言

この手番で行う行動を1つ宣言する。

### 4-2. カードプレイ

宣言した行動に対して、手札を**1枚**プレイする。  
手番が来たら、必ず手札を1枚使わなければならない。手札をプレイせずに行動することはできない。

手札が**0枚**になった場合、即座に山札から手札を**3枚**補充する。山札が尽きた場合は、捨て札をシャッフルして新しい山札を作る。

### 4-3. 行動段階決定

プレイしたカードの数字から、この手番の**行動段階**を決定する（「カードの見方」を参照）。

### 4-4. 行動解決

宣言した行動の処理を行う。対象、コスト、スート、効果、待機値の扱いは、各行動の項目に従う。

### 4-5. カウント変更

行動によって決定した待機値ぶん、自分のコマを後方へ進める。11を超える場合は次ラウンドの対応カウントに置く。

### 4-6. 手番終了

自分の手番終了時に発生する効果を処理する。手番終了処理が終わったら、そのキャラクターの手番は終了する。

## 5. カードの見方

戦闘中にプレイしたカードは、**数字**と**スート**を参照する。

### 5-1. 数字と行動段階

プレイしたカードの数字によって、その手番の**行動段階**が決まる。行動段階は、待機値や効果量を決めるために使用する。

| カードの数字 | A〜3 | 4〜7 | 8〜10 | J〜K |
| ------------ | ---- | ---- | ----- | ---- |
| 行動段階     | 軽行動 | 中行動 | 重行動 | 特行動 |

どの数字のカードでも、任意の行動を宣言できる。効果側で指定がない限り、カードの数字は行動の種類を制限しない。

### 5-2. スート

スートは、以下のマナ属性に対応する。

| スート   | スペード | ハート | ダイヤ | クラブ |
| -------- | -------- | ------ | ------ | ------ |
| 対応属性 | 火       | 水     | 土     | 風     |

スートの使い方は、各行動の項目に従う。効果側で指定がない限り、スートが一致していなくても行動自体は可能である。

## 6. 行動段階と待機値

行動段階は、行動後に進むカウント数を決めるために使用する。

### 6-1. 基本待機値

| 行動段階 | 軽行動 | 中行動 | 重行動 | 特行動 |
| -------- | ------ | ------ | ------ | ------ |
| 基本待機値 | 3    | 4      | 5      | 6      |

### 6-2. 重量補正

アイテムには重量が設定されており、使用したアイテムの重量によって待機値が変化する。

| 重量 | 軽量 | 中量 | 重量 |
| ---- | ---- | ---- | ---- |
| 重量補正 | −1 | ±0 | +1 |

**待機値 ＝ 行動段階の基本待機値 ＋ 使用したアイテムの重量補正**

アイテム・レシピを使用する場合、そのデータに設定された重量補正を用いる。使用するアイテムがない行動では、重量補正はない。

攻撃・防御・回避・離脱など、装備品を参照する行動では、`alchemy_and_items.md` の装備品テーブルに従い、行動段階と装備重量から待機値を決める。

## 7. 行動一覧

自身の手番では、以下の行動から1つを選んで宣言する。

| 行動 | 概要 |
| ---- | ---- |
| 攻撃 | 武器または素手を用いて、対象にダメージを与える |
| 防御 | 防具を用いて、ダメージに備える |
| 回避 | 攻撃や効果によるダメージを避ける準備をする |
| 分解 | アイテムや素材を分解し、マナを得る |
| 使用 | 所持しているアイテムを消費して、その効果を発動する |
| 想成 | レシピを参照し、マナを支払って効果を発動する |
| 離脱 | 戦闘から離れる準備をする |

### 攻撃の定義

**攻撃**は、武器または素手を用いて対象にダメージを与える行動のみを指す。  
レシピやアイテムでダメージを与えたとしても、それは攻撃として扱わない。

## 8. 攻撃

攻撃の待機値は、行動段階と武器の重量を用いて決定する（`alchemy_and_items.md` の武器テーブル参照）。

攻撃を行う場合、以下の順に処理する。

1. 攻撃指定
2. 属性確認
3. ダメージ算出
4. 攻撃効果適用
5. ダメージ処理

### 8-1. 攻撃指定

攻撃に使用する武器を1つ選び、攻撃対象を指定する。素手で攻撃できる場合は、素手を武器として扱う。

### 8-2. 属性確認

使用する武器の武器属性を確認する。武器にマナ属性が設定されている場合は、そのマナ属性も確認する。武器にマナ属性が設定されていない場合、その攻撃は無属性として扱う。

### 8-3. ダメージ算出

使用武器の品質、等級、攻撃係数を確認する。攻撃係数は、行動段階と武器重量で決まる。算出式は `alchemy_and_items.md` を参照する。

### 8-4. 攻撃効果適用

武器効果、印記、その他の攻撃側の効果を適用する。この段階で、対象に適用するダメージの値、属性、追加効果を確定する。

### 8-5. ダメージ処理

攻撃効果適用で確定したダメージを、対象に適用する。以降は「ダメージ処理」の項に従う。

## 9. ダメージ処理

ダメージ処理は、確定したダメージを対象に適用する処理である。  
ダメージの値、属性、追加効果は、発生元で確定する。  
ここでは、耐性、回避、**防護**（`<装甲>`、`<障壁>`、防御値など）など、受ける側の処理を扱う。

ダメージ処理を行う場合、以下の順に処理する。

1. 耐性適用
2. 回避処理
3. 防護処理
4. ダメージ適用
5. ダメージ後効果

### 9-1. 耐性適用

対象がダメージの属性に対応する耐性を持つ場合、その耐性を適用する。  
記述されていない耐性は**通常**として扱い、特別な処理は発生しない。  
同じ属性に対して、複数の耐性を同時に持つことはない。

#### 弱点

弱点の場合、そのダメージの追加効果として、ノックバック1とスタン値1を追加する。  
ただし、反射が適用される場合、弱点による追加効果は発生しない。

#### 半減

半減の場合、受けるダメージは半分（端数切り捨て）に減少する。

#### 無効

無効の場合、受けるダメージが0になる。ただし、ダメージ以外の追加効果は発生する。

#### 反射

反射の場合、そのダメージは対象ではなく、発生元に適用される。  
反射されたダメージには、回避、防護、追加効果を適用しない。  
発生元のHPを、反射されたダメージ分減少させる。  
反射が発生した場合、この対象へのダメージ処理は終了する。

### 9-2. 回避処理

対象が回避値を持つ場合、対象は1d10を振り、出目が現在の回避値以下なら回避に成功する。  
成功した場合、受けるダメージは0になり、そのダメージに付随する追加効果も発生しない。その後の処理は行わず、ダメージ処理を終了する。

### 9-3. 防護処理

対象が防護を持つ場合、以下の順に適用する。

1. `<装甲>`
2. `<障壁>`
3. 防御値

それぞれの防護は、現在のダメージをその値分軽減する。  
防護によって受けるダメージが0になった場合、以降の防護を適用しない。  
`<装甲>`でダメージを軽減した場合、軽減した値だけ強度を減らす。  
`<障壁>`の減衰処理は、`<障壁>`の効果に従う。

### 9-4. ダメージ適用

全ての軽減処理を終えた後、残った値を最終的に受けるダメージとする。  
対象のHPを、ダメージ分減少させる。HPが1点以上減少した場合、その対象にダメージを与えたものとして扱う。

### 9-5. ダメージ後効果

ダメージ処理の後に発生する効果を処理する。ノックバック、スタン値増加などは、この段階で処理する。

## 10. 属性

戦闘では、ダメージに設定された属性を参照する。属性には、**マナ属性**と**武器属性**がある。

### 10-1. マナ属性

マナ属性は、**火・土・風・水・闇・光**と**無**の7種類である。  
マナ属性が設定されていないダメージは、**無属性**として扱う。

### 10-2. 武器属性

武器属性は、**斬・打・突**の3種類である。  
攻撃によるダメージは、使用した武器に設定された武器属性を持つ。  
レシピやアイテムによるダメージは、武器属性が指定されていない限り、武器属性を持たない。

### 10-3. 複数属性を持つダメージ

ダメージが複数の属性を持つ場合、対象がそれぞれの属性に対して持つ耐性を確認する。

- 弱点に該当する属性がある場合、弱点1つにつき、ノックバック1とスタン値1を追加する。複数属性が弱点なら累積する
- 半減、無効、反射に該当する属性を確認し、最も優先度の高いものを1つだけ適用する
- 耐性の優先度は、低い順に **半減 → 無効 → 反射**
- 反射を適用する場合、弱点による追加効果は発生しない

## 11. 防御

防御には、**防御値**と**防御行動**がある。  
防御値は、ダメージを受けた時に常に参照される防護である。  
防御行動は、自分が受けるダメージに備えて`<装甲>`を得る行動である。

### 11-1. 防御値

防具を装備しているキャラクターは、防御値を持つ。防御値の算出は `alchemy_and_items.md` を参照する。

### 11-2. 防御行動

防御行動は、防具を用いて`<装甲>`を得る行動である。  
使用する防具を1つ選び、その防具の品質、等級、防御係数から`<装甲>`の強度を算出し、得る。  
防御係数と待機値は、行動段階と防具の重量によって決まる。

## 12. 回避

回避は、攻撃や効果によるダメージを避けるために`<回避>`を得る行動である。  
回避行動を行う場合、使用する防具を1つ選び、その防具の回避値を持つ`<回避>`を得る。  
回避値と待機値は `alchemy_and_items.md` の防具テーブルに従う。

- `<機敏>`などによって追加で得られる回避値は、最大5まで
- 回避値の合計は、9を上限とする

## 13. 分解

分解は、アイテムや素材を消費してマナを得る行動である。  
分解するアイテムまたは素材を1つ選び、消費する。選んだアイテムのタグに対応するマナを得る。  
さらに、プレイしたカードのスートに対応するマナを1つ得る。  
分解の待機値は、行動段階の基本待機値で決定する。

## 14. 使用

使用は、所持しているアイテムを消費して、その効果を発動する行動である。  
使用するアイテムを1つ選び、消費する。その後、アイテムの効果を処理する。  
ダメージが発生する場合、値・属性・追加効果を確定し、ダメージ処理を行う。  
使用の待機値は、行動段階とアイテムの重量で決まる。

## 15. 想成

想成は、レシピを参照して効果を発動する行動である。  
想成するレシピを1つ選び、レシピに書かれたマナを支払う。  
このときカードのスートと同じマナが含まれている場合、そのうち1つを無料にできる。  
その後、レシピの効果を処理する。ダメージが発生する場合はダメージ処理を行う。  
想成の待機値は、行動段階とレシピの重量で決まる。

## 16. 離脱

離脱は、戦闘をやめてその場を離れる行動である。

- 離脱を宣言したキャラクターは、次の自身の手番まで、回避処理や防護処理は行えない
- 次に自分の手番が来た時、戦闘から離脱する
- 離脱したキャラクターのコマをタイムラインから取り除く
- 一度離脱したキャラクターは、その戦闘に再参加できない
- ボス戦などの特殊な状況では、離脱を宣言することができない可能性がある
- 離脱の待機値は、行動段階と防具の重量によって求めた後に、**+2**する

## 17. ノックバック

ノックバックXは、その対象のコマをXカウント分後方へ進める効果である。  
カウントがタイムラインを周回する場合は、次のラウンドの対応する位置へ移る。

## 18. スタン

各キャラクターは**スタン値**と**スタン最大値**を持つ。  
PCのスタン最大値は5である（`character_creation.md` 参照）。敵のスタン最大値は、敵データに従う。

スタン値がスタン最大値以上になった場合、以下の処理を行う。

1. スタン値を0に戻す。この時、超過したスタン値は持ち越さない
2. 手札からランダムに1枚選ぶ
3. 選ばれたカードの数字に対応する行動段階の基本待機値分、ノックバックを受ける
4. 選ばれた手札を捨てる

スタン値は戦闘中に蓄積し、戦闘終了時に0へ戻る。

## 19. 戦闘不能

HPが0以下になったキャラクターは**戦闘不能**になる。  
戦闘不能になったキャラクターのコマは、タイムラインから取り除く。  
戦闘不能の間、そのキャラクターは戦闘に参加できない。

戦闘不能は、使用または想成などによって解除できる。  
解除されたキャラクターは、解除した味方の次の手番開始時から行動できる。  
戦闘不能になっても、所持しているマナは失われない。

## 20. 戦闘終了

以下のいずれかを満たした場合、戦闘は終了する。

- 敵のコマがタイムラインから全て取り除かれる
- PCのコマがタイムラインから全て取り除かれる
- 簡易戦闘のラウンド上限に達した
- 効果やイベントが戦闘終了を指定した

### 全滅

PC全員が戦闘不能になった場合、**全滅**として扱う。

1. 戦闘中なら、即座に戦闘を終了する
2. 素材カゴの素材を半分以上ランダムに失う
3. 探索に参加していたPC全員のHPを1にする
4. その時間の行動を終了し、街に帰還する

### 撤退

全滅ではなく、PCのコマがタイムラインから全て取り除かれた場合、**撤退**として扱う。撤退した場合、その戦闘には勝利しない。

### 勝利

敵のコマがすべて取り除かれた場合、**勝利**として扱う。  
簡易戦闘では、ラウンド上限まで生存した場合も勝利として扱う。

### 終了処理

残りの手札を捨て札に送る。特記がない限り、戦闘終了後は探索に戻る。

## 21. 簡易戦闘

探索中のイベントとして発生する戦闘は、常に簡易戦闘である。

- 簡易戦闘には、ラウンド数の上限がある
- 特に指定がなければ、**2ラウンド**終了時に戦闘終了となる
- 敵を全滅させるか、ラウンド上限まで生存すれば勝利となる
- 戦果は、倒した敵に応じて決定する

## 22. Joker

Jokerは戦闘中、**任意のスート・任意の数字**として扱える。  
使用時に、どのスート・どの数字として使うかを宣言する。
