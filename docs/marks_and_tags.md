# marks_and_tags.md

最終更新: 2026-06-22  
担当範囲: タグと印記の共通原則、記述方鷑、強度と解除の共通処理、参照関係、個別印記の本文  
このファイルに書かないもの: 戦闘の手番進行、スタンやノックバックの戦闘骨格（→ `combat.md`）、レシピ工程、敵データ

## 1. 共通原則

- 本作では、**タグ**と**印記**を別概念として扱う
- 名称表記は固定する
  - 印記: `<>`（例: `<毒>`、`<炎>`）
  - タグ: `[]`（例: `[鉱石]`）
- タグ/印記ともに、別名・省略名は認めない
- タグ名と印記名の同名衝突は認めない
- 命名は接頭辞を使わず、一覧管理で重複を防ぐ

### 記述方針

- 箇条書きの厳格テンプレートより、文章の流れをそろえることを優先する
- 必要な情報だけ書き、不要な項目は省略してよい
- ルールとデータは分離して管理する
- 一覧台帳（印記・タグ・アイテム等）の開発正本は [`catalog/`](../catalog/README.md)（SQLite）とする
- PL 提示用のミラーは Notion で行う

## 2. タグ

### 位置づけ

- タグはアイテムの素材的・フレーバー的な性質を示す分類情報である
- タグ自体はゲーム効果を持たない。マナ運用・素材選定・レシピ照合の判断基準として参照されることはある
- PCはタグ単体を保持しない

### 重複と抽出

- 同名タグの重複は許容しない
- 重複した場合は、1つ以外を余計なタグとして扱う
- タグだけを抜き出して保存する処理は、基本ルールから外す

### タグ記述のひな型（文章フロー）

1. タグ名を示す
2. タグが何を意味するかを書く
3. 必要な場合のみ、参照先効果や使用上の注意を書く

#### 文章ひな型（例）

- `[タグ名]：［意味/性質］を表す。［必要なら参照効果］。［必要なら注意事項］。`

#### 記述例

- `[鉱石]：採掘物・金属系の素材であることを示す。素材選定やレシピ照合での条件として参照される。`

## 3. 印記

### 位置づけ

- 印記は、名称で参照・解除・上書き判定ができる**効果の分類名**である
- 非印記効果は、付与側の効果文に持続/解除を書く

### 基本運用

- 同名印記が複数ある場合、適用されるのは最も強い効果1つのみ
- 弱い同名印記は消滅せず、規定タイミングまで残る
- 強い印記が切れたら、残っている同名印記のうち最も強いものが適用される
- 同時付与時の適用順は使用者が選ぶ
- 印記の有無・残り強度・残り時間は公開情報とする

### 強度・上限・耐性

- 強度の指定は任意。強度を持たない印記には以下の強度ルールは適用しない
- 強度0以下になった印記は解除される
- 印記耐性で強度が0以下になる場合は付与失敗
- 強度上限は必要な印記だけ個別に設定し、設定時は効果文に明記する
- 印記耐性処理は許可し、持続や適用タイミングは付与効果側で記述する

### 共通タイミング語

印記効果文で繰り返し使う減衰タイミングを以下の3語に統一する。

- **終了時減衰**: 自身の手番終了時に、その印記の強度を1減らす
- **発動時減衰**: その印記の効果が発動した直後に、その印記の強度を1減らす
- **開始時解除**: 次の自身の手番開始時に、その印記の強度を0にする

### 解除と失敗時処理

- 「解除不能」を固定タグ化せず、効果文で「いつ解除されるか」「それまでは解除できない」を書く
- 解除無効が既知ならコストは消費する
- 情報不足で不成立だった場合は、巻き戻し可否をGMが判断する

### 印記記述のひな型（文章フロー）

1. 印記名を示す
2. 効果の発動条件と起こる内容を書く
3. 必要な場合のみ、強度・持続・解除を追記する

#### 文章ひな型（例）

- `<印記名>：［必要なら発動条件］に［効果］する。［必要なら強度ルール］。［必要なら持続条件］。［必要なら解除条件］。`

#### 記述例

- `<毒>：自分の手番開始時に現在強度分のダメージを受ける。強度はこの処理の後に1減少し、0になったら解除される。『解毒薬』など、<毒>を解除すると明記された効果で解除できる。`

### 行動データでの使用形式

- 武器・アイテムの効果欄に直接 `<印記名>：〔効果〕` と書くことで、発動条件と効果を1行で示すことができる
- この場合の印記名は**トリガー条件の略式表現**として機能する。対象に状態を付与するのではなく、使用者が該当条件を満たした時に効果が発動することを意味する
- 例: `<重撃>：ノックバック+1` → 重行動帯カードで行動を宣言した時、ノックバック+1を与える
- トリガー条件として使う印記は台帳に登録し、発動条件を明記する

## 4. タグと印記の参照関係

- 「相互作用」は独立した特別ルールではなく、各効果文がタグ/印記を参照することを指す
- タグ参照で印記を付与する効果は制限しない
- 印記参照でタグを付与/変化させる効果も制限しない
- 効果文中でタグと印記を書く順番は固定しない
- ループ防止として、1解決内で同名効果は1回まで適用する
- 参照失敗時のコストは、既知失敗なら消費、未知失敗ならGM裁定とする
