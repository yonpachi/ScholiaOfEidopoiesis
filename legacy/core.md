# core.md

最終更新: 2026-03-22  
担当範囲: ゲーム全体の基本原則と共通前提  
このファイルに書かないもの: フェイズ手順、探索手順、戦闘手順、個別データ

## 1. ゲームの中心

本作は、町と工房を拠点に日々を積み重ねる錬金術TRPGである。  
大きな区切りは通常のTRPG的なシナリオではなく、**一日ごとの進行**に置く。

基本的な循環は次の通り。

**町 / 工房の発展 → 新地景の開放 → 探索 → 素材獲得 → 分解 / 作成 / 納品 → 町 / 工房の発展**

## 2. プレイ単位

- 一日の終わりごとに、次の日を続けるかどうかを決める
- あえて言えば、**一日 = 1プレイ単位**として扱う
- 町と工房は、日ごとに少しずつ成長していく

## 3. 目指す体験

本作で重視するのは以下である。

- 日々の暮らしや工房運営
- 依頼処理と納品
- 素材を持ち帰り、分解し、作成する反復
- 少しずつ町と工房が広がっていく感覚
- 参加人数や担当者が変わっても続けやすい継続遊戯

世界を救う一本道の大筋を最初から強く置く必要はない。  
重い要素や物語は、主依頼や長期計画として後ろに置いてよい。

## 4. 目標の層

本作の目標は、日々・中期・長期の三層で積み上がる。

### 日々の目標

- 納品依頼を達成する
- 必要な品を作る
- 小さな依頼をこなす
- 素材を集める

### 中期的な目標

- 工房ランクを上げる
- 設備を整える
- 倉庫を広げる
- 新しい地景を開く
- 新しいレシピや作成物を増やす

### 長期的な目標

- 大きな依頼や古い謎を進める
- 遠方の土地を開く
- 町全体の規模や文化を変えていく

## 5. 継続キャンペーンとして共有されるもの

本作で継続される主体は、PC個人だけではない。  
特に次のものが共有財産として積み重なる。

- 工房
- 町
- NPC
- 依頼掲示板
- 倉庫
- 工房ランク
- 設備
- 解放された地景とマップ
- 長期計画

## 6. ルール設計の基本方針

- 紙でも運用できることを優先する
- 電子補助は便利にするためのものであり、電子でしか成立しない構造は避ける
- データより先に、各場面で何ができるかの裁定を揃える
- 実際に卓を回さないと分からない部分は、無理に確定しない
- 同じ決定事項を複数の文書へ書かない

## 7. 管理単位

本作では**アイテムが基本単位**である。

- PCはタグ単体を保持しない
- タグはアイテムに付属する情報である
- マナは、素材やアイテムのタグに基づいて使う
- 探索で持ち帰るのは実物アイテムや素材である

## 8. タグの原則

- 同名タグの重複は許容しない
- 重複した場合、1つ以外は余計なタグとして扱う
- 良い素材とは、品質値が高い素材ではなく、**必要なタグを持つ素材**である
- タグと印記の共通定義・記述規則・参照関係の詳細は `marks_and_tags.md` を正本とする

## 9. ルールで強く規定するもの / 創作を許すもの

### ルール側で強く規定したいもの

- 判定の基本構造
- 探索の流れ
- 戦闘の骨格
- 錬金の骨格
- 地景や敵データの書式
- 依頼・工房・町成長の接続

### GMや卓ごとに増やしてよいもの

- 依頼本文
- 小さなNPC
- 町の細部設定
- 日々の出来事
- 個別レシピや個別アイテム

## 10. 保留の置き方

次のような内容は、現時点では無理に確定しない。

- 数値バランス
- 戦闘内の細かな補正
- 地景1つごとの個別Jokerテンプレート
- エンカウント表の粒度
- 状態異常や個別スキルの詳細

これらは、実データ作成やテストプレイを通して確定させる。  
この姿勢自体が本作の基本方針である。
